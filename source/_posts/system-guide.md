## 欢迎使用宋青的博客系统

本博客已经成功升级为高能华丽的 Hexo + 安知鱼主题系统。在此，你拥有完全自主的文章撰写、管理和部署权限。为了让你流畅开始，本指南将为你介绍如何在这个全新的“数据大仓”里发布你自己的文章。

### 第一步：创建新文章 📝

因为我们配置了 **GitHub Actions 自动化部署流水线**，你不再需要通过命令行或复杂的软件写文章，甚至可以不需要本地 Node.js 环境：
1. 直接在你的 GitHub 仓库中，导航到 `source/_posts/` 目录。
2. 点击右上角的 **Add file** -> **Create new file**（新建文件）。
3. 命名你的文件为 `文章英文名.md`（例如 `my-first-post.md`）。

### 第二步：编写 Markdown 语法与 Front Matter ✏️

在新建的文件中，头部**必须**包含如下格式的元数据（被称为 Front Matter），用于定义文章标题、日期、分类和标签：

```yaml
---
title: 我的第一篇安知鱼文章
date: 2026-05-31 10:00:00
categories:
  - 技术
tags:
  - 体验
  - 极客
---
```

在元数据下方，你可以使用标准的 Markdown 语法自由撰写内容：
- **标题**：使用 `#`、`##`、`###`。
- **列表**：使用 `-` 或 `*` 表示无序列表，`1.` 表示有序列表。
- **代码块**：使用三个反引号（\`\`\`）并指定语言，例如 \`\`\`javascript。
- **高亮卡片与提示**：安知鱼主题支持极其丰富的高级标签（Tag Plugins），如 `{% note success %} 成功提示 {% endnote %}` 等。

### 第三步：提交并自动云端发布 🚀

1. 编写完成后，滑动到页面底部，点击 **Commit changes...** 提交按钮，将文件保存到 `main` 分支。
2. 提交保存后，**GitHub Actions 将在几秒钟内在云端自动启动构建**！
3. 它会自动拉取你的新文章，编译成超炫的安知鱼页面，并在 1-2 分钟内将更新自动推送到你的自定义域名 `www.songqing.online` 上！
